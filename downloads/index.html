There are plenty of ways to get Stratosphere. Pick any of the following to start.

Ready to Run Packages

Download the ready to run binary package if you want to use Stratosphere on your computer or cluster.

If you are planning to use Stratosphere with data stored in HDFS or HBase or if you want to run it using YARN, choose a Stratosphere version that matches your Hadoop version. Stratosphere ships with code to access Hadoop (e.g. HDFS, HBase and YARN). If you want to use Stratosphere without Hadoop, we recommend using the Hadoop 1 enabled version.

Maven Dependencies

Use the Maven Dependencies to add Stratosphere to your project.

These dependencies also include a local execution environment and thus support local testing.

Adding the entries below to your project's dependencies (in the pom.xml file) causes Maven to include the Stratosphere classes in your project. If you want to interact with Hadoop (HDFS), pick the dependency matching your Hadoop version. In doubt, use the Stratosphere version for Hadoop 1.x.

  • Hadoop 1
  • Hadoop 2
{% highlight xml %} eu.stratosphere stratosphere-java {{ site.current_stable }} eu.stratosphere stratosphere-clients {{ site.current_stable }} {% endhighlight %}
{% highlight xml %} eu.stratosphere stratosphere-java {{ site.current_stable_yarn }} eu.stratosphere stratosphere-clients {{ site.current_stable_yarn }} {% endhighlight %}

The listed dependencies reference the Java API. To use the Scala API, replace the stratosphere-java artifact id with stratosphere-scala.

If you don't have a Maven prodeject yet, we encourage you to use our Quickstart scripts to create one with all the Stratosphere dependencies already in place.

Debian Package

We have also prepared a Debian repository for Debian/Ubuntu systems.

{% highlight bash %} sudo vim /etc/apt/sources.list.d/stratosphere.list # deb http://dev.stratosphere.eu/apt/ ./ wget -q http://dev.stratosphere.eu/apt/stratosphere.gpg -O- | sudo apt-key add - sudo apt-get update sudo apt-get install stratosphere-dist {% endhighlight %}

To start the services run:

{% highlight bash %} sudo service taskmanager start sudo service jobmanager start {% endhighlight %}

Source

Compile Stratosphere on your own machine.

  • Hadoop 1
  • Hadoop 2
{% highlight bash %} git clone https://github.com/stratosphere/stratosphere.git cd stratosphere mvn clean package -DskipTests {% endhighlight %}
{% highlight bash %} git clone https://github.com/stratosphere/stratosphere.git cd stratosphere mvn clean package -DskipTests -Dhadoop.profile=2 {% endhighlight %}

Make sure to checkout the Documentation for further help.

Nightly Builds

The Nightly Builds contain the most recent development code. Use them for example if you need a feature before its release. Only builds that pass all tests are published here.

SNAPSHOT-Versions in Sonatype Maven

Add the Sonatype Snapshots repository to into your Maven pom: {% highlight xml %} snapshots-repo https://oss.sonatype.org/content/repositories/snapshots false true {% endhighlight %} Set the version to the next snapshot version, for example: 0.5-SNAPSHOT {% highlight xml %} eu.stratosphere stratosphere-clients {{ site.current_snapshot }} {% endhighlight %}
